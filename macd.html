Wstęp¶

Tematem projektu byo zapoznanie si z teoria MACD, a nastpnie implementacja tego wskanika. Do wyboru byy dwa jzyki: Python i c++. W swoim projekcie posuyem si tym pierwszym. Dane giedowe jakich uyem to kwoty zamknicia WIG20, gdzie najbardziej aktualny kurs pochodzi z 01-03-2019.

Krótkie omówienie wskaźnika¶

Wskanik przedstawiany jest w formie dwch linii. Linii MACD i wyznaczonej na jej podstawie linii sygnau. Linia MACD tworzona jest na podstawie rnicy wykadniczej redniej kroczcej, dugookresowej i krtkookresowej (w tym przypadku 26 i 12 okresowej). rednie obliczane s z poniszego wzoru:


$$EMA_n = \frac{p_0+(1-\alpha)p_1+(1-\alpha)^2p_2+...+(1-\alpha)^Np_N}{1+(1-\alpha)+(1-\alpha)^2+...+(1-\alpha)^N}$$
$$\alpha = \frac{2}{N+1}$$

def ema(n, sample_set):
    alfa = 2/(n+1)
    up = sum(map(lambda x,y: x*(1-alfa)**y, sample_set, range(n)))
    down = sum(map(lambda x: (1-alfa)**x, range(n)))
    result = up/down
    return result

def macd(n, sample_set):
    result = []
    samples = list(reversed(sample_set))
    for i in range(n):
        result.append(ema(12, samples[i:12+i]) - ema(26, samples[i:26+i]))
    return list(reversed(result))

def signal(n, sample_set):
    result = []
    samples = list(reversed(sample_set))
    for i in range(n):
        result.append(ema(9, samples[i:9+i]))
    return list(reversed(result))

Przydatność wskaźnika MACD w analizie technicznej:¶

Przecicie si lini MACD i SIGNAL sygnalizuje moment, w ktrym nastpuje zmiana trendu (momenty kupna i sprzeday), niestety sygnay te s czsto sygnaami spnionymi i nie nadaj si do inwestycji krtkoterminowych (z powyszych wykresw wynika, e nawet mae zmiany trendu mog mie horrendalny wpyw na zmian wskanika). Gdy nastpio przecicie i linia SIGNAL znajduj si nad lini MACD, generowany jest sygna sprzeday, gdy sytuacja jest odwrotna sygna kupna. Wskanik oczywicie najlepsze zastosowanie znajduje w inwestycjach dugoterminowych oraz nadaje sido wskazania stanu wykupienia rynku (gdy linie znajduj si wysoko ponad lini zera lub bardzo nisko). Zauwaalana jest take dywergencja czyli rozbieno w zachowaniu ceny i wskanika, ktra poprawnie interpretowana moe by wskazwk wiadczc o nadchodzcej zmianie trendu. Oddlego linii MACD i SIGNAL wskazuje sie trendu, a najlepiej obrazuje to histogram MACD. Podsumowujc wskanik ten zdecydowanie moe by pomocny w analizie technicznej, w dugoterminowych inwestycjach, jeeli uywany jest jako wskazwka i przy podejmowaniu jakichkolwiek decyzji nie s bran pod uwag tylko i wyacznie generowane sygnay.

Algorytm symulujący¶

Algorytm podejmuje dezycje kupna i sprzeday na podstawie sygnay generowanego przez wskanik MACD (sprawdzane jest wystpienie przecicia i jego kierunek), nastpnie w zalenoci od sygnau sprzedawane s wszystkie akcje lub kupowana jest taka ilo akcji na jakie s rodki (w celu lepszego zobrazowania profitu oraz podejmowanych decyzji dopuszczalna jest moliwo kupowania czci akcji). Na przedstawionym poniej wykresie markery na lini sygnaowej oznaczaj moment kupna, na lini MACD, sprzeday.

import random

def simulation(data_set, money, simul_time, plot=""):
    start_money = money
    stocks = 0.0
    macd_list = macd(simul_time, data_set)
    signal_list = signal(simul_time, macd_list)
    value_list = list(reversed(data_set)) #date order in data_set is descending, starting from recent date
    buy_signals = []
    sell_signals = []
    for day in range(1, simul_time-1):
        sell = macd_list[day] > signal_list[day]
        buy = macd_list[day] < signal_list[day]
        sell_cross = macd_list[day-1] < signal_list[day-1]
        buy_cross = macd_list[day-1] > signal_list[day-1]
        crossed = (buy and buy_cross) or (sell and sell_cross)
    
        if buy and crossed and value_list[day] > 0 and money>0:
            ammount = money
            stocks += ammount/value_list[day]
            money -= ammount
            buy_signals.append(day)
            
        elif sell and crossed and value_list[day] > 0 and stocks>0:
            ammount = stocks
            stocks -= ammount
            money += ammount*value_list[day]
            sell_signals.append(day)
            
    money += stocks*value_list[simul_time - 1]
    profit = money/start_money*100.0 -100.0
    if plot == "print_plot":
        print("starting ammount: " + str(start_money) + " current ammount: " + str(money) + " profit " + str(profit) + "%")
        plt.plot(macd_list, '-ro', markevery=sell_signals, label="MACD")
        plt.plot(signal_list, '-b*', markevery=buy_signals, label="SIGNAL")
        plt.rcParams['figure.figsize'] = 10, 5
        plt.title("SIMUL CHART")
        plt.grid(True)
        plt.legend()
        plt.show()
    elif plot == "print_summary":
        print("profit: "+ str(profit))
    return profit

Rezultat symulacji¶

Dla 1000 prbek uzyskany profit to ok. 8.1%, aczkolwiek po przeprowadzeniu symulacji dla 100-1300 prbek (co 100) moemy stwierdzi, e algorytm nie jest algorytmem optymalnym. Na zachwianie siwynikw znaczcy wpyw ma prostota w podejmowaniu decyzji tj. brak analizowania dywergrencji i siy trendu. Ilo kupowanych i sprzedawanych akcji jest staa, a nie zalena od siy trendu. Z powyszych wynikw symulacji mona take zauway, e dla wiekszj iloci prbek osigany profit jest zdecyowanie wyszy, lecz wystpuj te strate spowodowane np. faszywymi sygnaami ze wzgldu na dywergencje.
